# dig-node 控制面板

DIG Chrome extension 内置了一个用于管理本地 dig-node 的**控制面板**。通过它,你可以查看节点的实时状态、决定为缓存内容预留多少磁盘空间,并且——完成一次性配对步骤后——管理节点的上游、其托管的存储库、同步状态以及对等节点。日常使用无需命令行。

控制面板是 DIG Browser 节点管理界面在扩展程序中的内置对应功能:它与运行在你自己机器上的节点通信,因此一切都保留在本地。

## 打开它

1. 打开扩展程序。
2. 进入 **Network**(网络)标签页,选择 **Control**(控制)。(紧凑弹出窗口只显示摘要;使用**打开控制面板**可查看每个部分的全屏视图。)

面板会自动检测节点:

- **节点正在运行** → 你会看到管理视图。
- **未找到节点** → 你会看到一个关于如何安装节点的简短页面。浏览功能仍可正常使用——内容读取会回退到公共网络;只有管理视图才需要节点。

## 实时状态

顶部的实时指示器会显示你的节点处于**已连接**、**正在连接**还是**已断开**状态,并显示其地址和版本。它会自动更新——启动或停止节点后,指示器会在几秒内翻转,无需重新打开面板或刷新页面。

## 为缓存内容预留磁盘空间(缓存与 LRU)

你的节点会在本地缓存已获取的内容,这样重复访问会更快,同时你也在帮助分发这些内容。缓存有一个**预留大小**——即它可以使用的磁盘空间上限。当缓存超过该上限时,节点会自动优先移除**最近最少使用**的项目("LRU" 策略),因此你预留的空间永远不会被超出,而你实际使用的内容会持续保留在缓存中。

此部分无需配对即可立即使用。

**查看已用空间。** 一个进度条会显示已用空间相对于预留上限的比例,以及若干实时数据:已缓存的项目数量、它们的总大小、自节点启动以来已淘汰的量,以及缓存命中/未命中次数。

**设置预留上限。** 输入新的大小并应用。最小值为 **64 MiB**;更小的值会被提升到该下限。将上限降低到低于当前已用空间时,会触发淘汰最旧的项目,直到用量符合要求为止。

**查看并移除缓存项目。** 缓存列表会显示每个项目的大小、上次使用时间及其**淘汰顺序**(位置 `0` 表示下一个将被移除的项目)。你可以:

- **淘汰单个项目** — 立即移除单个缓存项目。
- **清空全部** — 清空整个缓存。

移除项目只会释放本地磁盘空间;你再次访问的任何内容都会被重新获取。

:::tip
如果这是一台你经常用来浏览的机器,请为缓存分配尽可能多的空间——更大的预留空间意味着更少的重新获取,以及更多内容在本地提供服务。在空间有限的机器上,设置较小的预留值;LRU 会保留最有用的项目并丢弃其余项目。
:::

## 管理节点(需要配对)

其余各部分会更改节点的配置,因此需要你的明确授权。由于扩展程序运行在浏览器的沙盒中,它无法直接读取节点的本地权限文件——因此你需要先**配对**一次。配对会为扩展程序授予其自己的、范围受限且可撤销的凭据;它永远不会暴露节点的主密钥,并且只能在运行该节点的计算机上批准。

### 将扩展程序与你的节点配对

1. 在控制面板中,选择**配对**。扩展程序会显示一个 **6 位数代码**和一个配对 ID。
2. 在运行该节点的计算机上的终端里,运行 `dig-node pair` 以列出待处理的请求(或直接运行 `dig-node pair approve <pairing-id>`)。
3. 确认终端中显示的代码与扩展程序中的代码**一致**,然后批准。这种匹配是你的安全保障:它确保你批准的是*这个*扩展程序,而非其他任何东西。
4. 控制面板会自动切换到已配对状态。该凭据仅由扩展程序存储。

配对代码会**在几分钟后过期**;如果你在批准之前代码已过期,请再次选择**配对**以获取新代码。

要停止使用该凭据,请在面板中选择**取消配对**(这只会在本地忘记它)。要在节点本身上撤销它,请在该计算机上运行 `dig-node pair revoke <token-id>`——面板会在下一次操作时返回到未配对状态。

:::note
配对仅是下面管理部分所需要的。上面的实时状态与缓存/LRU 控制无需配对即可使用。
:::

### 上游

查看节点获取内容所用的上游,并可设置不同的上游。更改后的上游会在节点下次启动时生效。

### 托管的存储库

查看你的节点持有并固定的存储库,固定一个新的存储库(让节点保留并提供服务)、取消固定某个存储库,并检查任意存储库的状态。固定特定版本会预先获取它,以便随时可以提供服务。

### 同步

查看是否可以进行经过身份验证的整库同步,并针对特定版本触发同步,使节点获取并缓存它。

### 对等节点

查看你节点的对等网络状态——它与中继的连接情况(用于在家庭路由器背后实现可达性),以及它当前连接的对等节点。
