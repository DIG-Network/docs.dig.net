# dig-node 控制面板

DIG Chrome extension 內建了一個管理本機 dig-node 的**控制面板**。透過它,你可以查看節點的即時狀態、決定要為快取內容保留多少磁碟空間,並且——在完成一次性配對步驟後——管理節點的上游、其託管的儲存庫、同步狀態以及對等節點。日常使用不需要命令列。

控制面板是 DIG Browser 節點管理畫面在擴充功能中的內建對應功能:它會與執行在你自己機器上的節點通訊,因此一切都保留在本機。

## 開啟它

1. 開啟擴充功能。
2. 前往 **Network**(網路)分頁,選擇 **Control**(控制)。(精簡彈出視窗僅顯示摘要;使用**開啟控制面板**可檢視每個區塊的全螢幕畫面。)

面板會自動偵測節點:

- **節點執行中** → 你會看到管理畫面。
- **找不到節點** → 你會看到一個說明如何安裝節點的簡短頁面。瀏覽功能仍可正常使用——內容讀取會回退到公共網路;只有管理畫面才需要節點。

## 即時狀態

頂部的即時指示器會顯示你的節點處於**已連線**、**連線中**還是**已中斷連線**狀態,並顯示其位址與版本。它會自動更新——啟動或停止節點後,指示器會在數秒內翻轉,不需要重新開啟面板或重新整理頁面。

## 為快取內容保留磁碟空間(快取與 LRU)

你的節點會在本機快取已擷取的內容,讓重複造訪更快,同時你也在協助分發這些內容。快取有一個**保留大小**——也就是它可以使用的磁碟空間上限。當快取超過該上限時,節點會自動優先移除**最近最少使用**的項目(「LRU」策略),因此你保留的空間永遠不會被超出,而你實際使用的內容會持續留在快取中。

此區塊不需要配對即可立即使用。

**查看已用空間。** 一個進度條會顯示已用空間相對於保留上限的比例,以及幾項即時數據:已快取的項目數量、其總大小、自節點啟動以來已淘汰的量,以及快取命中/未命中次數。

**設定保留上限。** 輸入新的大小並套用。最小值為 **64 MiB**;較小的值會被提升到該下限。將上限降低到低於目前用量時,會觸發淘汰最舊的項目,直到用量符合要求為止。

**檢視並移除快取項目。** 快取清單會顯示每個項目的大小、上次使用時間及其**淘汰順序**(位置 `0` 表示下一個會被移除的項目)。你可以:

- **淘汰單一項目** — 立即移除單一快取項目。
- **全部清除** — 清空整個快取。

移除項目只會釋放本機磁碟空間;你再次造訪的任何內容都會被重新擷取。

:::tip
如果這是一台你經常用來瀏覽的機器,請為快取分配盡可能多的空間——較大的保留空間代表較少的重新擷取,以及更多內容在本機提供服務。在空間有限的機器上,設定較小的保留值;LRU 會保留最有用的項目並捨棄其餘項目。
:::

## 管理節點(需要配對)

其餘各區塊會變更節點的設定,因此需要你的明確授權。由於擴充功能是在瀏覽器的沙盒中執行,它無法直接讀取節點的本機權限檔案——因此你需要先**配對**一次。配對會授予擴充功能它自己的、範圍受限且可撤銷的憑證;它永遠不會揭露節點的主金鑰,並且只能在執行該節點的電腦上核准。

### 將擴充功能與你的節點配對

1. 在控制面板中,選擇**配對**。擴充功能會顯示一組 **6 位數代碼**與一個配對 ID。
2. 在執行該節點的電腦上開啟終端機,執行 `dig-node pair` 以列出待處理的請求(或直接執行 `dig-node pair approve <pairing-id>`)。
3. 確認終端機中顯示的代碼與擴充功能中的代碼**相符**,然後核准。這項比對是你的安全保障:它確保你核准的是*這個*擴充功能,而非其他任何東西。
4. 控制面板會自動切換到已配對狀態。憑證僅由擴充功能儲存。

配對代碼會**在幾分鐘後過期**;若你在核准之前代碼已過期,請再次選擇**配對**以取得新代碼。

若要停止使用該憑證,請在面板中選擇**取消配對**(這只會在本機忘記它)。若要在節點本身撤銷它,請在該電腦上執行 `dig-node pair revoke <token-id>`——面板會在下一次動作時回到未配對狀態。

:::note
配對僅是下方管理區塊所需要的。上方的即時狀態與快取/LRU 控制不需要配對即可使用。
:::

### 上游

檢視節點擷取內容所用的上游,並可設定不同的上游。變更後的上游會在節點下次啟動時生效。

### 託管的儲存庫

檢視你節點持有並固定的儲存庫,固定一個新的儲存庫(讓節點保留並提供服務)、取消固定某個儲存庫,並檢查任何儲存庫的狀態。固定特定版本會預先擷取它,以便隨時可以提供服務。

### 同步

檢視是否可以進行已驗證的整庫同步,並針對特定版本觸發同步,使節點擷取並快取它。

### 對等節點

檢視你節點的對等網路狀態——它與中繼的連線狀況(用於在家用路由器後方達成可達性),以及它目前連線的對等節點。
